# ISO 42001 Exam Speed Sheet (Open-Book)

## 1) 10-Second Navigation Map

| If Question Mentions... | Go To | Quick Note |
|---|---|---|
| Context, internal/external issues | Clause 4.1 | Use SWOT/PESTIL only if relevant to AIMS purpose |
| Interested parties, needs, expectations | Clause 4.2 | "Relevant" parties only, not everyone |
| Scope boundaries/exclusions | Clause 4.3 | Context first, scope second |
| Leadership/accountability | Clause 5.1 | Top management accountability cannot be delegated |
| Policy | Clause 5.2 | Policy = direction/framework, not control checklist |
| Roles/responsibilities | Clause 5.3 | Named owners + communicated authority |
| Risk assessment/treatment | Clause 6.1 | Criteria first, then assess and treat |
| SoA | Clause 6.1 + Annex A | All controls listed, selected/excluded with reason |
| Objectives/KPIs | Clause 6.2 | Measurable + monitored + aligned to policy |
| Competence/awareness/docs | Clause 7 | Evidence required, not assumptions |
| Operational control/change | Clause 8.1 | Run what was planned |
| AI impact assessment | Clause 8.2 | Distinct from risk assessment |
| Monitoring/audit/review | Clause 9 | Check what works and what does not |
| NC, correction, CA, improvement | Clause 10 | Root cause is key for corrective action |
| Control selection guidance | Annex A/B | A = controls, B = implementation guidance |
| Objective ideas | Annex C | Use as objective/KPI source |
| Sector/context tailoring | Annex D | Use for proportional interpretation |

## 2) Scenario Decision Rules (Use in MCQ/Scenario Items)

1. If policy exists but no top-management ownership, then answer is weak/nonconforming (Clause 5).
2. If scope was defined before context/interested parties, flag sequence error (Clauses 4.1 -> 4.2 -> 4.3).
3. If objectives have no metric/method/owner/frequency, they are not complete (Clause 6.2).
4. If SoA excludes controls without rationale, it is nonconforming (Clause 6.1 + Annex A).
5. If risk and impact assessment are treated as one process with no distinction, flag as incorrect (6.1 vs 8.2).
6. If monitoring is vanity-only (no decision value), it is poor implementation (Clause 9.1).
7. If management review has no top management participation, it does not satisfy Clause 9.3.
8. If only immediate fix is done with no cause analysis, corrective action is missing (Clause 10.2).
9. If internal audit is done by the same person who owns the process, objectivity is compromised (Clause 9.2).
10. If third-party AI is used without transparency/provenance checks, escalate as supply-chain risk (Clause 6 + Annex A domains).

## 3) Confusion Killers (One-Liners)

- Risk assessment: organizational and system risks that affect AIMS outcomes (Clause 6.1).
- Impact assessment: consequences of specific AI systems on people/society/context (Clause 8.2).
- Correction: immediate containment/fix.
- Corrective action: remove root cause to prevent recurrence.
- Policy: leadership direction and commitments.
- Objective: measurable target derived from policy and risk/impact outputs.
- Internal audit: independent check of conformity/effectiveness.
- Management review: strategic evaluation by top management.

## 4) High-Yield Pitfalls

- Scope before context.
- Policy drafted without leadership ownership.
- Stakeholder list without relevance filter.
- Objectives without measurable method.
- SoA treated as an operations checklist.
- No distinction between correction and corrective action.
- No evidence trail from context -> risk -> control -> monitoring -> improvement.

## 5) 3-Hour Exam Pacing (Adjust to Actual Duration)

- Pass 1 (about 60% time): answer direct items fast, flag hard items.
- Pass 2 (about 30% time): solve flagged scenario questions using clause map.
- Pass 3 (about 10% time): re-check absolutes (always/never), ownership, and evidence logic.

## 6) Fast Evidence Words (Use in Explanations)

- Scope statement
- Interested parties register
- AI policy
- Roles/responsibility matrix
- Risk assessment criteria + risk register
- Statement of Applicability (SoA)
- AI system impact assessment record
- KPI/monitoring dashboard
- Internal audit program/report
- Management review minutes/actions
- Nonconformity + corrective action record

## 7) Last-Line Answer Template (When Unsure)

"Start from Clause 4 context and relevant interested parties, derive risks in Clause 6, select/justify controls via SoA, operate under Clause 8, verify through Clause 9 evidence, and close gaps through Clause 10 corrective action and continual improvement."

## 8) Final Reminder Sticky Notes

- Context before scope.
- Policy before objectives.
- Criteria before risk scoring.
- Risk before treatment.
- Evidence before assurance.
- Root cause before closure.
