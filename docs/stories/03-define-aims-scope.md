# User Story: 3 - Define AIMS Scope

**As a** AIMS implementation lead,
**I want** to define the scope of the AI management system,
**so that** the organization has a clear boundary for what is governed, audited, and certified.

## Acceptance Criteria

* The scope states the organizational boundaries, AI activities, and relevant systems covered by the AIMS.
* Scope exclusions are documented with justification.
* The scope reflects the documented context and interested party requirements.
* The approved scope is available for use in policy, risk, audit, and certification activities.

## Notes

* Directly derived from the trainer's point that scope cannot be defined in isolation and should follow understanding of context.
